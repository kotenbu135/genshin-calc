## ボトルネック: 条件付き効果の表現力不足

現在の `StatBuff` / `PassiveEffect` は**常時付与の固定値バフ**しか表現できない。
武器174本・聖遺物59効果が空になっている主因。

### 表現できない効果の例

- **HP閾値**: 護摩の杖「HP50%以下でATK+1.0% of Max HP」
- **元素反応トリガー**: 燃え盛る炎の魔女4pc「蒸発・溶解反応倍率+15%」
- **スタック効果**: 渡火4pc「烈晶反応後8秒間炎ダメ+40%、3スタック」
- **武器種限定**: 剣闘士4pc「片手剣・両手剣・長柄キャラのみ通常攻撃+35%」
- **敵側デバフ**: Zhongliシールド「全元素耐性・物理耐性-20%」
- **EM依存バフ**: Kazuha A4「EM×0.04%の元素ダメバフ」(現在はbuilder側で手動計算)

## TODO項目

### P0: 条件付き効果システムの設計

**目的**: 武器・聖遺物・天賦の条件付き効果を統一的に表現できるデータ構造を設計する。

**課題**:
- `ConditionalBuff` のような条件付きバフ型の設計
- 条件の種類: HP閾値、元素反応後、スタック数、武器種限定、キャラ元素限定
- core crateとdata crateの責務分離を維持
- WASM互換性の維持（動的ディスパッチ最小化）
- イミュータブル設計の維持

**設計選択肢** (要検討):
1. enum-based条件 (`BuffCondition::HpBelow(0.5)` 等)
2. タグベース (`tags: &[EffectTag]` でフィルタ)
3. ユーザー側で条件判定、data crateは最大値のみ提供

**影響範囲**: core (`buff_types.rs`, `team.rs`), data (`buff.rs`, 全武器/聖遺物/天賦)

### P1: 天賦バフ拡充 (既存枠内)

**目的**: 現在の `TalentBuffDef` で表現可能なサポートキャラを追加する。

条件付き効果システム不要で追加可能なキャラ:

| キャラ | バフ内容 | 備考 |
|--------|----------|------|
| Sucrose | EM共有 (A4: チームにEM×20%) | base_value方式 |
| Nilou | 開花反応ボーナス | 特殊条件あり(水草限定) → P0必要かも |
| Kuki Shinobu | (サポートとしてのバフは少ない) | スキップ |
| Gorou | DEFバフ (スキル: 岩キャラ数依存) | 条件付き → P0必要 |
| Lisa | DEF-15%デバフ (A4) | 敵側デバフ → 要設計 |
| Zhongli | 全耐性-20% (シールド) | 敵側デバフ → 要設計 |
| Chevreuse | 過負荷後に炎/雷耐性-40% | 条件付き + 敵側 → P0必要 |
| Faruzan | 風ダメバフ + 風耐性シュレッド | 一部P1対応可能 |
| Chiori | (サブDPS、チームバフ少ない) | スキップ |

**P1で即追加可能** (条件なし or builder側計算):
- Sucrose (EM共有)
- Faruzan (風ダメバフ部分のみ)

### P2: 聖遺物セット効果の充実

**目的**: 空になっている59効果のうち、使用頻度が高いものから `StatBuff` を埋める。

#### 2pc効果 (単純バフ — ほぼ全て実装済み)
2pc効果は大半が無条件バフのため、既に高カバレッジ。残りを確認して埋める。

#### 4pc効果 (条件付き — P0依存)
使用頻度が高い4pcセット:

| セット | 4pc効果 | 条件 |
|--------|---------|------|
| 燃え盛る炎の魔女 | 蒸発/溶解+15%, 過負荷等+40% | 反応トリガー |
| 剣闘士のフィナーレ | 通常攻撃+35% | 武器種限定 |
| 大地を流浪する楽団 | 重撃+35% | 武器種限定 |
| しめ縄の記憶 | 通常/重撃/落下+50% | スキル使用後 |
| 絶縁の旗印 | 爆発ダメ+25%(ER×) | ER依存 |
| 金メッキの夢 | EM/ATK%(チーム元素依存) | チーム構成依存 |
| 花海甘露の光 | 通常/重撃+10%~40%(HP変動) | HP変動スタック |

→ **武器種限定** (剣闘士, 楽団) は比較的単純な条件で、P0の初期ターゲットに最適

### P3: 武器パッシブの充実

**目的**: 174本の空パッシブのうち、人気武器から `StatBuff` を埋める。

#### 無条件パッシブ (即対応可能)
一部の武器は常時効果だが `StatBuff` が空のまま。確認して埋める。

#### 条件付きパッシブ (P0依存)
人気武器の優先リスト:

| 武器 | パッシブ | 条件 |
|------|---------|------|
| 護摩の杖 | HP基準ATKバフ | HP閾値 |
| 霧切の廻光 | 元素ダメ+12~28%(3スタック) | スタック |
| 磐岩結緑 | HP基準ATKバフ | 無条件(HP%) |
| 狼の末路 | ATK+20% + チームATK+40%(HP<30%) | HP閾値 + チーム |
| 千岩古剣 | ATK+20% 璃月キャラ数依存 | チーム構成 |
| 赤砂の杖 | EM基準ATK% | EM依存 |

### P4: 武器精錬値 (R1-R5)

**目的**: `refinement_values: Option<[f64; 5]>` フィールドにR1-R5の値を入力する。

**優先度**: 低。まずP0で条件付き効果が使えるようになってからでないと、精錬値を入れても計算に反映されない武器が大半。

P1-P3で `StatBuff` が実装された武器から順にR1-R5値を埋めていく。

### P5: 敵データ拡充

**目的**: 週ボス、最新ボスを追加。

追加候補:
- 週ボス: 風魔龍トワリン、タルタリヤ、若陀龍王、シニョーラ、雷電将軍(鍛錬)、散兵、アペプ、ナルヴァレット
- フィールドボス: 各種レジスヴァイン(雷/水/草)、マグーケンキ、恒常キューブ(氷/雷/草)
- 精鋭: アビスの使徒/詠唱者、遺跡ドレイク、エルマイト旅団

**既存の枠組みで即追加可能** — 耐性テンプレート + `EnemyData` を追加するだけ。P0不要。

### P6: 敵側デバフシステム

**目的**: Zhongli耐性シュレッド、超電導物理耐性-40%等を表現する仕組み。

現在の `Enemy.resistance` は単一f64値。元素別の耐性値を個別に持つか、耐性減少バフを `ResolvedBuff` に統合するか設計が必要。

## 推奨実装順序

1. **P5 敵データ** — 即追加可能、既存API変更なし、ウォームアップに最適
2. **P1 天賦バフ** — Sucrose, Faruzan等を既存枠で追加
3. **P0 条件付き効果** — 設計ドキュメント作成 → core変更 → data対応
4. **P2 聖遺物4pc** — P0完了後、武器種限定から着手
5. **P3 武器パッシブ** — P0完了後、人気武器から着手
6. **P4 精錬値** — P3と並行で対応
7. **P6 敵側デバフ** — P0の設計に含めるか別途検討
